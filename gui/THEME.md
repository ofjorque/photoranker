# Dirección de diseño — PhotoRanker GUI

Ver checklist de `docs/fase5-gui.md`. Decisiones concretas, no "moderno" en abstracto. Desde la migración a React + Tailwind + shadcn/ui, los tokens viven en `src/index.css` (bloques `:root`/`.dark`), extendidos por `tailwind.config.js` — ya no en `src/theme/dark.css`/`light.css`/`base.css` (esos archivos no existen más).

- **Tipografía**: Inter Variable (`@fontsource-variable/inter`, bundleado — sin CDN, la app funciona offline), configurada como `fontFamily.sans` en `tailwind.config.js`. Números tabulares (`font-variant-numeric: tabular-nums`, aplicado globalmente en `body`) para `mu`/`sigma`/BIC, que se re-renderizan en vivo y no deben "bailar" de ancho.
- **Paleta**: acento violeta (`#7c6fff` oscuro / `#6952e0` claro) — frío, no compite con las miniaturas de fotos que dominan la pantalla. Vive en `--primary` (`src/index.css`, formato `H S% L%` sin `hsl()`, ver `fase5-gui.md` "Theming"), no en un token propio — así todos los componentes de shadcn (que ya leen `--primary`) lo respetan automáticamente.
- **Espaciado**: escala de Tailwind por defecto (múltiplos de 4px vía `gap-*`/`p-*`/`m-*`) — nunca un padding "al ojo".
- **Profundidad ("glossy" en oscuro, agregado por feedback de uso real)**: el tema oscuro usa un stack de sombras multicapa con un highlight superior sutil (`inset 0 1px 0 rgba(255,255,255,...)` apilado sobre la sombra de caída) en vez de bordes de 1px, superficies con gradiente diagonal sutil (clases utilitarias `.bg-panel-gradient`/`.bg-panel-gradient-raised`), un glow ambiental muy tenue detrás de todo el shell (`--gradient-ambient` sobre `body`), y vidrio esmerilado (clase `.glass`, `backdrop-filter: blur(var(--glass-blur))`) en los overlays que flotan directamente sobre fotos (`ConfirmDialog`, `Lightbox`, `LoadingOverlay`) — ver `docs/fase5-gui.md`, sección "Dirección 'glossy' del tema oscuro", para la lista completa de tokens. **El tema claro se mantiene deliberadamente simple/plano por contraste** (mismos nombres de token, valores casi imperceptibles, sin el highlight `inset`) — el pedido de "glossy" fue específicamente para oscuro; no "arreglar" el bloque `:root` para que se vea igual de brillante.
- **Controles**: componentes de shadcn/ui (`src/components/ui/`, estilo "new-york") — nunca el look nativo del navegador. `--radius` (`src/index.css`) controla el radio base que consumen `rounded-sm/md/lg` en `tailwind.config.js`.
- **Elemento distintivo**: la transición de foco entre miniaturas del torneo — el borde de foco (`--ring`) se anima con `transition` + un halo (`box-shadow` difuso, clase utilitaria `.focus-halo`) que crece/decrece al mover el foco, en vez de aparecer/desaparecer abruptamente; en el tema oscuro se suma `--shadow-glow-accent` (un glow ancho adicional) para reforzar el efecto "glossy" sin reemplazar el halo original. El badge de empate (`bg-tie-badge`) tiene un pulso sutil (clase `.animate-tie-pulse`, `@keyframes tie-pulse` en `src/index.css`, vía `--tie-badge-rgb` para que el color del glow siga siendo correcto por tema) para diferenciarse a simple vista de un badge de posición asignada normal (`bg-success`).
- **Ventana sin decoración nativa**: `data-tauri-drag-region` + una barra de título propia (`src/components/TitleBar.tsx`) reemplazan la barra de Windows por defecto — ver `docs/fase5-gui.md`.

## Mecanismo de override (`theme_path`)

El tema embebido (clase `.dark` presente o ausente en `<html>`, según `config.toml: theme`) se aplica al arrancar; si `theme_path` apunta a un `.css` legible, su contenido se inyecta después, en un `<style>` separado — así solo necesita redefinir las variables que le interesan (típicamente solo `--primary`, generado por la pantalla de Ajustes a partir del color elegido). Fallback silencioso (sin romper la app) si el archivo no existe o no se puede leer — ver `src/theme/index.ts`.
