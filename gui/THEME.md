# Dirección de diseño — PhotoRanker GUI

Ver checklist de `docs/fase5-gui.md`. Decisiones concretas, no "moderno" en abstracto:

- **Tipografía**: Inter Variable (`@fontsource-variable/inter`, bundleado — sin CDN, la app funciona offline). Pesos 400/500/600/700 vía el eje variable. Números tabulares (`font-variant-numeric: tabular-nums`) para `mu`/`sigma`/BIC, que se re-renderizan en vivo y no deben "bailar" de ancho.
- **Paleta**: la del tema embebido oscuro/claro exigido por `fase5-gui.md` (tokens exactos en `src/theme/dark.css`/`light.css`). Acento violeta (`#7c6fff` oscuro / `#6952e0` claro) — frío, no compite con las miniaturas de fotos que dominan la pantalla.
- **Espaciado**: escala de 8px (`--spacing-unit`), múltiplos (4/8/16/24/32) — nunca un padding "al ojo".
- **Profundidad ("glossy" en oscuro, agregado por feedback de uso real)**: el tema oscuro usa un stack de sombras multicapa con un highlight superior sutil (`inset 0 1px 0 rgba(255,255,255,...)` apilado sobre la sombra de caída en `--shadow-sm/md/lg`) en vez de bordes de 1px, superficies con gradiente diagonal sutil (`--gradient-surface`/`--gradient-surface-raised`, reemplazan el fill plano de `.panel`/`.panel-nested`), un glow ambiental muy tenue detrás de todo el shell (`--gradient-ambient` sobre `body`), y vidrio esmerilado (`--glass-bg` + `backdrop-filter: blur(var(--glass-blur))`) en los overlays que flotan directamente sobre fotos (`ConfirmDialog`, `Lightbox`, `LoadingOverlay`) — ver `docs/fase5-gui.md`, sección "Dirección 'glossy' del tema oscuro", para la lista completa de tokens. **El tema claro se mantiene deliberadamente simple/plano por contraste** (mismos nombres de token, valores casi imperceptibles, sin el highlight `inset`) — el pedido de "glossy" fue específicamente para oscuro; no "arreglar" `light.css` para que se vea igual de brillante.
- **Controles**: `<input>`/`<select>`/botones con estilo custom completo (radios `--radius-md`, sin apariencia nativa del navegador) — ver `src/theme/base.css`.
- **Elemento distintivo**: la transición de foco entre miniaturas del torneo — el borde de foco (`--color-focus-border`) se anima con `transition` + un halo (`box-shadow` difuso del mismo color) que crece/decrece al mover el foco, en vez de aparecer/desaparecer abruptamente; en el tema oscuro se suma `--shadow-glow-accent` (un glow ancho adicional) para reforzar el efecto "glossy" sin reemplazar el halo original. El badge de empate (`--color-tie-badge`) tiene un pulso sutil (`animation`, vía `--color-tie-badge-rgb` para que el color del glow siga siendo correcto por tema) para diferenciarse a simple vista de un badge de posición asignada normal.

## Mecanismo de override (`theme_path`)

El tema embebido (`dark.css` o `light.css`, según `config.toml: theme`) se inyecta primero como un `<style>`; si `theme_path` apunta a un `.css` legible, su contenido se inyecta después, en un segundo `<style>` — así solo necesita redefinir las variables que le interesan. Fallback silencioso (sin romper la app) si el archivo no existe o no se puede leer — ver `src/theme/index.ts`.
